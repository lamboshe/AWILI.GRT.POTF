# Introduction to Git and GitHub

A beginner-friendly guide to help you use GitHub effectively in your learning journey — no prior programming experience required.

---

## What is Git?

Git is a version control system — it tracks every change you make to your files so you can go back to any previous version.

Created by Linus Torvalds in 2005, Git lets you:

- Save snapshots of your work (called commits)
- Experiment on branches without breaking your project
- Collaborate with others by merging changes

---

## What is GitHub?

GitHub is a website that hosts your Git projects online.

Why it matters for learning:

- Your projects are backed up online
- You build a public portfolio
- You learn the workflow used by professional developers

---

## Key Concepts

| Term | Meaning |
|-----|--------|
| Repository (Repo) | Folder tracked by Git |
| Commit | Saved snapshot of your changes |
| Branch | Separate line of work |
| Merge | Combining branches |
| Clone | Downloading a repo |
| Fork | Copying someone else's repo |
| Pull Request | Request to merge changes |
| Push | Upload commits to GitHub |
| Pull | Download updates |

---

## Basic Git Workflow

```bash
# Clone a repository
git clone https://github.com/your-username/your-repo.git

# Move into the folder
cd your-repo

# Check changes
git status

# Stage files
git add .

# Commit changes
git commit -m "Add introduction"

# Push to GitHub
git push
